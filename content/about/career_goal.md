# 커리어 목표

## 큰 방향

부서 간 데이터 사일로(Data Silo)로 같은 데이터를 서로 다르게 해석하거나 접근조차 못하는 문제를 실무에서 직접 목격한 이후, **비개발 직군도 셀프서비스로 필요한 데이터에 접근할 수 있는 데이터 민주화(Data Democratization) 플랫폼**을 만드는 데이터 엔지니어가 되는 것이 장기 지향점이다.

## 단기 목표 (1~2년)

### 기술 확장
- **실시간 처리 스택 실무 경험**: Kafka Streams 또는 Spark Structured Streaming을 이용한 실시간 파이프라인 구축
- **대용량 분산 처리 경험**: 테라바이트 단위 데이터를 Spark로 처리하는 실제 업무
- **데이터 플랫폼 관점의 설계**: 개별 DAG 단위가 아닌 메타데이터 관리·카탈로그·권한까지 포함한 플랫폼 단위 설계

### 조직
- 데이터 플랫폼팀 또는 데이터 인프라팀에서 시니어 엔지니어와 함께 일하며 플랫폼 설계 경험 축적
- 개인 프로젝트(게임 챗봇, Text-to-SQL)에서 다룬 LLM 기반 데이터 접근 기술을 실무 제품에 녹여보고 싶음

## 중기 목표 (3~5년)

### [작성 필요: 중기 목표]
예시:
- "데이터 플랫폼 엔지니어로서 한 팀의 플랫폼 기능 설계·운영을 리드"
- "Text-to-SQL·BI 자동화 등 데이터 접근성 개선 제품을 담당"
- "Airflow 기반 오케스트레이션 대체 스택(Dagster, Prefect 등) 실무 도입 경험"

- "OSS 기여 또는 자체 도구 출시"

## 관심 있는 도메인

### 현재 경험 도메인
- 축산 방역 / 공공 데이터 (검역본부)
- 부동산 / 금융 데이터 (고객사 납품)
- 경매 데이터 (외부 판매용)
- 게임 데이터 (TFT, 로스트아크 — 개인 프로젝트)

### [작성 필요: 다음에 경험하고 싶은 도메인]
예: "커머스(실시간 주문·재고)", "핀테크(결제 트랜잭션)", "물류", "미디어 로그 분석" 등

## 피하고 싶은 환경

### [작성 필요: 합류 시 고려 요소]
면접 "어떤 회사를 찾으시나요?" 질문 대응용. 예:
- "데이터 엔지니어가 1명뿐이라 멘토링 받기 어려운 환경"
- "기술 부채만 처리하고 새 플랫폼 설계 기회가 없는 역할"
- "비즈니스와 단절된 순수 인프라 업무만 하는 조직" 등
