# 지원동기 / 회사 선호도

> 데이터 엔지니어 직무를 선택한 배경과, 어떤 회사·팀·환경을 찾고 있는지를 정리.
> 답변은 1분 내(3~6문장).
> 회사 특정 지원동기는 면접 직전 회사별로 추가 작성 필요.

---

## 데이터 엔지니어 진로 동기

### 1차 계기 — 학부 수업
CUHK 클라우드 컴퓨팅 수업에서 **넷플릭스의 페타바이트급 데이터 처리 사례**를 접한 것이 계기였습니다. 단순히 데이터를 저장하는 게 아니라, 분산된 시스템에서 데이터를 쪼개고 합치는 과정 자체가 흥미로웠고, **데이터의 흐름을 설계하는 사람**이 되고 싶다고 생각하게 되었습니다.

### 학부 시절 검증
- CUHK 빅데이터 수업에서 MapReduce 기반 K-means, SON, PCY 알고리즘을 직접 구현
- Hadoop 분산 시스템 설계 프로젝트 (개인 포트폴리오 — `hadoop-distributed-system`)
- 해당 경험을 통해 "이론으로 흥미를 느낀 영역이 실제 구현에서도 재미있다"는 것을 확인

### 실무 진입 후 가치 확인
실무에서는 **검역본부 공무원들이 본인이 만든 파이프라인을 신뢰하며 사용하는 모습**, **분석가·개발팀이 설계한 테이블 구조를 이해하고 활용하는 모습**을 보며 이 일의 가치를 확신하게 되었습니다. 데이터 엔지니어는 사용자 한 명 한 명이 매번 데이터를 직접 다루지 않아도 되도록 **인프라 단에서 신뢰성을 보장하는 역할**이라는 점이 매력적입니다.

## 선호하는 회사·팀 환경

### 가장 중요하게 보는 3가지

#### 1. 시니어 DE에게 멘토링 받을 수 있는 환경
1년차로서 아직 배워야 할 영역(실시간 처리, 대용량 분산, 플랫폼 설계, IaC, 데이터 거버넌스 등)이 명확하기 때문에, **데이터 엔지니어가 본인 혼자가 아닌 팀** 환경을 우선 봅니다. 코드 리뷰·설계 리뷰·페어 프로그래밍이 가능한 조직.

#### 2. 데이터 플랫폼 관점의 설계 기회
개별 DAG 단위가 아니라 **메타데이터 관리·카탈로그·권한·셀프서비스**까지 포함한 플랫폼 단위 사고를 할 기회가 있는 팀. 실무에서 개별 파이프라인은 충분히 다뤘기 때문에, 다음 단계는 플랫폼 관점이라고 보고 있습니다.

#### 3. 비즈니스와 연결된 데이터 조직
순수 인프라만 다루는 조직보다, **비즈니스 임팩트와 연결되는 데이터 작업**이 가능한 환경을 선호합니다. 검역본부·부동산처럼 사용자가 명확하고, 만든 결과물이 실제 의사결정에 쓰이는 환경에서 일했던 경험이 만족도가 높았기 때문입니다.

### 기술 스택 선호도

[수정 필요: 본인 선호도 — 자유롭게 조정]
- **선호**: Airflow / Kafka / Spark / dbt / Snowflake·BigQuery 같은 모던 데이터 스택
- **관심**: Dagster, Prefect 등 차세대 오케스트레이션 / Iceberg, Delta Lake 같은 lakehouse 포맷
- **개의치 않음**: 클라우드 벤더(AWS·GCP·Azure 모두 학습 의지 있음)

## 피하고 싶은 환경

### [수정 필요: 솔직하되 면접에서 부정적으로 들리지 않게]

예시 방향:
- "데이터 엔지니어가 1명뿐이라 멘토링 받기 어려운 환경 — 1년차에게는 학습 환경이 중요해서"
- "기존 파이프라인 유지보수만 하고 새 시스템 설계 기회가 없는 역할"
- "데이터 거버넌스·품질에 대한 인식이 낮아 단순 적재만 요구하는 조직"
- "비즈니스와 단절된 순수 인프라 업무만 하는 조직"

## 입사 후 기여 방향

### 단기 (입사 0~6개월)
- 빠른 도메인 학습 — 비개발 직군과의 소통 경험을 살려 도메인 담당자·시니어 DE 양쪽으로부터 빠르게 흡수
- 기존 파이프라인 코드·설계 문서 정독 후 개선 포인트 제안
- 본인 강점 영역(Airflow 운영, 데이터 검수 체계)에서 즉시 기여

### 중기 (6개월~1년)
- 새 스택(실시간 처리, 데이터 플랫폼 도구)을 실무에 적용하며 학습
- 본인 책임의 도메인·파이프라인을 가지고 운영
- 코드 리뷰·설계 토론에 적극 참여하면서 시니어로부터 흡수

### [수정 필요: 회사·직무별 추가 기여 방향]
면접 직전 해당 회사의 채용 공고와 매칭하여 구체화.

## 회사별 지원동기 (면접 직전 작성)

### 작성 템플릿

회사별로 별도 작성. 다음 4가지 요소를 포함:

1. **회사·산업 매력 포인트**
   "OO 산업의 OO한 특성에 매력을 느꼈고, 특히 [구체적 제품·서비스]가 인상적이었습니다."

2. **직무 매력 포인트**
   "지원한 [팀명]은 [구체적 기술 스택·도전 과제]를 다루고 있어, 제가 다음 단계로 성장하고 싶은 영역과 일치합니다."

3. **본인 경험 연결**
   "저는 빅밸류에서 [관련 경험]을 통해 [관련 역량]을 키웠고, 개인 프로젝트로 [관련 영역]을 학습했습니다."

4. **기여 의지**
   "입사 후 [단기 기여] + [중기 학습 방향]으로 팀에 기여하고 싶습니다."
