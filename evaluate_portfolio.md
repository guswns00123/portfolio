# 포트폴리오 종합 평가

> 평가 관점: 데이터 엔지니어 채용 결정자 (시니어 DE / 팀 리드)
> 평가 대상: 주니어 데이터 엔지니어 유현준
> 평가일: 2026-04-05

## Overall: B+ (주니어 기준)

> "기본기 있고 실무 경험도 괜찮다. 면접은 보겠다. 다만 몇 가지 아쉬운 점이 있다."

---

## 좋은 점 (면접까지 가는 이유)

### 1. 실무 경험의 폭이 주니어치고 넓다

검역본부(폐쇄망/예측모델), 부동산 시세(B2B 납품/QA), 경매(크롤링/테이블 설계), GIS(공간 데이터) — 1년 안에 이 정도 도메인 다양성은 꽤 인상적. "시키면 뭐든 한다"는 인상을 줌.

### 2. 정량적 성과가 있다

12x 개선, 4.6배 확장 — 숫자가 있으면 일단 읽게 됨. 주니어 중에 성과를 수치로 표현할 줄 아는 사람이 많지 않음.

### 3. "구축"뿐 아니라 "운영" 경험

파이프라인 만들고 끝이 아니라 일별 운영, 장애 대응, QA 프로세스까지 경험한 건 확실한 플러스.

---

## 아쉬운 점 (경쟁자와 비교했을 때)

### 1. 개인 프로젝트가 약하다 — 가장 큰 약점

솔직히 TFT, 로또 프로젝트는 "Airflow 학습용"이라는 느낌이 강함. 문제는:

- MDX 상세 내용이 너무 얇음 (개요 3줄 + 아키텍처 나열이 전부)
- **"왜 이 기술을 선택했는지"**, **"어떤 문제를 만나서 어떻게 해결했는지"**가 전혀 없음
- GitHub 링크도 없어서 코드를 확인할 수 없음

실무 프로젝트는 기밀이라 간략하게 쓰는 게 맞지만, **개인 프로젝트마저 얇으면 "이 사람이 진짜 깊이 있게 한 건가?"라는 의문**이 생김. 개인 프로젝트야말로 구체적으로 써야 할 곳.

### 2. Strengths 섹션이 자소서 톤

"능동적 탐구력", "경청과 소통" — 채용 담당자는 이런 추상적 키워드를 거의 안 읽음. 포트폴리오는 자소서가 아니라 **"증거를 보여주는 곳"**. 강점은 Experience와 Projects에서 자연스럽게 드러나야지, 별도 섹션으로 "나는 소통을 잘합니다"라고 쓰면 오히려 설득력이 떨어짐.

### 3. About 섹션과 Experience 섹션 내용이 겹침

"12배 단축", "4.6배 확장"이 Stats, About, Experience에서 3번 반복됨. 같은 얘기를 3번 하면 "이것밖에 없나?" 하는 인상. Stats에서 한 번 임팩트를 주고, Experience에서 디테일을 보여주면 충분.

### 4. 기술 스택이 전부 비슷함

4개 실무 프로젝트 태그가 거의 다 `Python, Airflow, PostgreSQL, SQL`. 이건 사실이니까 바꿀 수는 없지만, "이 사람은 이것만 할 줄 아나?"라는 인상을 줄 수 있음. 개인 프로젝트에서 **다른 기술 깊이**를 보여줘야 밸런스가 맞음. (예: Spark 깊이 있게, Docker/K8s 경험, CI/CD, 테스트 코드 등)

## 구체적 개선 제안 (우선순위순)

| 순위 | 항목 | 이유 |
|------|------|------|
| 1 | **개인 프로젝트 MDX 내용 대폭 보강** | "문제 → 시도 → 실패 → 해결" 스토리텔링. 코드 스니펫, 아키텍처 다이어그램, 성능 비교 등 |
| 2 | **GitHub 링크 연결** | 코드 없는 포트폴리오는 신뢰도가 반으로 떨어짐 |
| 3 | **Strengths 섹션 제거 또는 변경** | 차라리 "Certifications & Education"으로 바꾸거나, 강점은 프로젝트 서술 안에서 자연스럽게 보여주기 |
| 4 | **About 섹션 간결화** | Stats에서 이미 숫자를 보여줬으니, About은 "나는 누구이고 무엇을 지향하는가" 2-3줄이면 충분 |
| 5 | **이미지 placeholder 교체** | 빈 placeholder가 있으면 "미완성" 인상. 당장 이미지가 없으면 placeholder 자체를 빼는 게 나음 |

## 한 줄 요약

> **실무 경험은 주니어치고 확실히 강점인데, 개인 프로젝트의 깊이와 코드 증거가 부족해서 "이 사람의 실력을 직접 확인"할 방법이 없다.** 면접에서 물어보면 잘 답할 것 같은데, 서류 단계에서는 아쉬움이 남는 포트폴리오.
