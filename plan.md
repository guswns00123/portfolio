# Experience 섹션 카드 구성 Plan

## 공통 카드 구조

각 카드는 포트폴리오 페이지 내에서 완결되며, 상세 페이지로 이동하지 않음.

```
[제목]
[소속 · 기간]
[한 줄 요약]
[아키텍처 한 줄] (기술 흐름만 간략히)
· 핵심 성과 1
· 핵심 성과 2
[기술 스택 태그]
```

---

## 카드 1: 질병 예측 데이터 파이프라인

- **제목**: 검역본부 가축 질병 예측 데이터 파이프라인
- **소속 · 기간**: 빅밸류 · 2025-06 ~ 2025-12
- **한 줄 요약**: 폐쇄망 환경에서 ASF/HPAI/FMD 예측 모델용 데이터 전처리·적재 파이프라인 구축 및 운영
- **아키텍처**: Oracle → PostgreSQL → Airflow → 예측 모델 → PostgreSQL → UI
- **핵심 성과**:
  - 레거시 산출 프로세스 처리 속도 12x 개선
  - 원격 운영을 위한 테이블별 task 분리 및 장애 대응 로그 체계 설계
- **기술 스택**: Python, Airflow, PostgreSQL, Oracle, SQL

---

## 카드 2: 부동산 시세 데이터 파이프라인

- **제목**: 부동산 시세 데이터 파이프라인
- **소속 · 기간**: 빅밸류 · 2025-04 ~ 2026-02
- **한 줄 요약**: 금융권 고객사 대상 월별 시세 데이터 생성·검수·납품 파이프라인 운영
- **아키텍처**: 공공데이터 + 자체 데이터 + 고객사 데이터 → Airflow → 검수 → CSV 납품
- **핵심 성과**:
  - 기존 레거시 코드 개선 및 프로세스 속도 개선 및 로직 정확도 향상
  - 전월 대비 변동률·이상치 자동 탐지 검수 체계 구축
  - 이상값 발생 시 원인 추적까지 포함한 QA 프로세스 운영
- **기술 스택**: Python, Airflow, PostgreSQL, SQL

---

## 카드 3: 경매 데이터 수집 파이프라인

- **제목**: 경매 데이터 수집 파이프라인
- **소속 · 기간**: 빅밸류 · 2025-04 ~ 2026-02
- **한 줄 요약**: 외부 판매용 경매 데이터의 자동 수집·가공 파이프라인 구축
- **아키텍처**: 웹 크롤링 → Python 정제 → PostgreSQL (비정규화 테이블)
- **핵심 성과**:
  - 분석가가 조인 없이 바로 활용 가능한 비정규화 테이블 설계
  - 안정적 수집을 위한 재시도 로직 및 누락 알럿 구성
- **기술 스택**: Python, Airflow, PostgreSQL, Web Crawling

---

## 카드 4: 도로 교차로 네트워크 구축

- **제목**: 도로 교차로 네트워크 구축
- **소속 · 기간**: 빅밸류 · 2025-03 ~ 2025-04
- **한 줄 요약**: 사내 분석팀용 전국 도로 교차로 공간 데이터 확장 및 속성 설계
- **아키텍처**: 정부 도로 데이터 → QGIS 교차점 추출 → Python 후처리 → PostgreSQL
- **핵심 성과**:
  - 교차점 데이터 약 4.6배 확장 (골목길 수준 커버리지 확보)
  - 이동 방향 수, 군집 ID 등 분석용 공간 속성 추가 설계
- **기술 스택**: Python, QGIS, PostgreSQL, GIS

## 참고: 보안 가이드라인

- 고객사 실명 미노출 (은행명, 회계회사명 등 → "금융권 고객사", "외부 고객사")
- 구체적 데이터 건수 미노출 → 비율/배수로 표현 (12x, 4.6배)
- 내부 시스템명, DB명, 테이블명 미노출
- 아키텍처는 기술 흐름 수준만 (구체적 서버 구성 등 제외)

## 추가되거나 검토 바람 사항:
질병 예측 데이터 파이프 라인에 핵심 성과로 결국 검역본부 고객사에서 안정적으로 해당 프로그램을 이용하였고 매일 안정적 데이터파이프라인 운영을 제공하고 지속적으로 고객사 대응(데이터 측면)을 맡았어. 그리고 데이터 사이언티스트, 개발팀과의 지속적인 소통으로 이게 가능했음.

처리 속도 개선에 대한 구체적인 수치가 더 들어가면 좋을듯 

그다음 교차로 데이터 업무에서는 
교차점 데이터 약 4.6배 확장 (골목길 수준 커버리지 확보) -> 이거에 대한 구체적인 수치가 있으면 좋겠음
  - 이동 방향 수, 군집 ID 등 분석용 공간 속성 추가 설계 -> 이거로 인해서 데이터 분석가들이 해당 데이터들을 더 잘 활용할 수있게끔 데이터 확장성을 높임. 이러한게 들어갔으면 좋을지도
